4 / Implementation notes

How the page was built

High-signal implementation choices
  • Used OKLCH colors with tinted neutrals for a more intentional palette.
  • Paired Fraunces for display with Instrument Sans for body copy.
  • Kept the whole landing page as a single portable HTML file with inline CSS.
  • Structured the page into hero → proof → features → testimonial → CTA for easy evaluation.
:root {
  --bg: oklch(13% 0.03 250);
  --text: oklch(95% 0.015 88);
  --accent: oklch(81% 0.17 94);
}

.hero {
  display: grid;
  grid-template-columns: 1.15fr 0.85fr;
  gap: 2rem;
}

h1 {
  font-family: "Fraunces", serif;
  font-size: clamp(3.4rem, 8vw, 7rem);
}

6 / Risks & caveats

What this does not prove

Not production-wired No real app behavior, data loading, analytics, or backend integration. This is a visual front-end artifact only.
External dependency Fonts are loaded from Google Fonts, so a fully offline render could differ without bundling font files.
Limited QA scope I verified structure and screenshots, but did not run a dedicated accessibility pass, cross-browser comparison, or performance audit.
